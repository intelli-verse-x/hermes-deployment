Platform wiring — what the Agency stands on

The four AI Agency codebases don't work alone. They plug into the intelli-verse-x platform: a model gateway, a work-tracking fabric, a content engine, and the same MCP servers that power the admin portal copilot. This page maps those connections.

The 5-layer stack

LayerServicesJob
1 · StrategyLeantime, GitHubWhat humans care about: goals, tickets, code work.
2 · CoordinationBeads (bd), BernsteinWhat agents do: the issue graph and each project's autonomous crew.
3 · Multi-projectGas Town (rigs, polecats, Mayor)The fabric that spans every repo — dispatching, convoys, mail.
4 · PlatformLiteLLM, Infisical, LangfuseModel gateway (one budget, any provider), secrets, observability.
5 · Gluen8n, GitHub ActionsWorkflow automation and CI.

The AI Agency (Hermes) sits on top of all five as the user-facing conversational layer: it translates plain-English asks into bead-tracked work and dispatches through the right layer. It is deliberately not the router between specialist agents — Gas Town's Mayor and Beads own that.

Model access — LiteLLM only

  • Hermes is configured with provider: custom pointing at the LiteLLM gateway. Model switching (hermes model) needs no code or key changes.
  • Budgets and keys are per-team virtual keys; traces flow to Langfuse automatically.
  • The same gateway serves the admin copilot, Content Factory and every other service — see the AI Platform Docs (LiteLLM).

Tool access — MCP everywhere

Every capability is an MCP server; the agent's priority order (from config/AGENTS.md):

Generate video/image/audiomcp_content_factory_*Content Factory (50+ pipelines)

Tool-bloat rule The agent doesn't keep every server enabled forever — finished with a Content Factory run, it drops those tools (/toolset, mcp_disable_server). Context is a budget.

Relationship to the admin portal copilot

The AI Agency and the admin portal copilot are siblings, not the same thing:

Admin portal copilotAI Agency (Hermes)
Lives inThe browser (portal chat)A 24/7 gateway process (cluster or laptop)
Reaches tools viaadmin-mcp gateway (~48 downstream MCP servers)Direct MCP client connections from config.yaml
MemoryPer-conversationPersistent: session search, curated memory, user modeling
SkillsPortal skill catalog (Skills Guide)agentskills.io mounts (7 repos)
Best forQuick reads and gated writes across the stackLong-running, learning, scheduled, multi-channel work

They share the underlying services (LiteLLM, the MCP servers, the skill philosophy), so a capability added for one is usually a config edit away for the other.
