Desktop apps — two shells, two jobs

The AI Agency has two desktop surfaces. They look similar (a native window with our tools inside) but they wrap different things: one wraps the hosted admin portal, the other wraps the Hermes agent itself. Neither reimplements features — both follow the "fork the surface, not the runtime" rule.

1 · Intelliverse X (Tauri shell)

SEPARATE REPO  intelliverse-x-desktop — "desktop shell for the Intelliverse admin portal (Windows, macOS, Linux)".

PropertyValue
FrameworkTauri 2 (Rust core + system webview — tiny binaries, no bundled Chromium)
Product name / idIntelliverse X / ai.intelli-verse-x.desktop
What it showsThe hosted portal at admin.intelli-verse-x.ai, loaded directly via WebviewUrl::External in src-tauri/src/lib.rs. The local ui/index.html is only a loading placeholder.
Layoutsrc-tauri/ (Rust: window, permissions, capabilities, icons) · ui/ (placeholder page) · scripts/
Buildnpm run dev / npm run build (Tauri CLI v2)
Why this design Because the window loads the live portal, the desktop app is always up to date — every portal deploy (copilot, tiles, skills, insights) ships to desktop users instantly, with no app-store release cycle. The Rust shell only adds what the web can't: a real window, OS integration, and a place to grow native capabilities (tray, notifications, deep links).

2 · Hermes Desktop (Electron client, inside the fork)

LIVES IN hermes-agent/desktop  The upstream Hermes Desktop client (merged upstream May 2026), customized in our fork as the "IX Agency admin portal shell with VPN, MCP and Skills panels".

PropertyValue
FrameworkElectron 38 + TypeScript, bundled with esbuild (scripts/build.mjs)
Packagehermes-desktop 0.1.0 (private), main: dist/main/main.js
What it wrapsThe Hermes gateway — connect to a local or remote agent with a session token and get the full dashboard/TUI experience in a window.
Our panelsVPN (connect to the private network the agent operates in), MCP (inspect/enable tool servers), Skills (browse and trigger skills).
Pipelinenpm run generate → typecheck → bundle; npm start to run; --smoke flag for CI smoke tests.

Which one is "the AI Agency app"?

Intelliverse X (Tauri)Hermes Desktop (Electron)
AudienceEveryone on the team — admins, creators, opsAgent power-users and engineers
WrapsHosted admin portal (copilot, tiles, insights)The Hermes agent gateway directly
Needs a server?No — just internetYes — a local or remote Hermes gateway
UpdatesInstant (web deploys)App release (npm build)
StatusEarly — repo scaffolded, window loads production portalActive — tracks the fork's release train

Long-term the two converge: the Tauri shell is the distribution vehicle (small, signed, auto-updating), and the agent panels from Hermes Desktop graduate into portal pages, which the shell then picks up for free.
