agent-skills — the shared skill registry

CANONICAL REGISTRY   intelli-verse-x/agent-skills (checkout: intelli-verse-x-agent-skills). A skill is a markdown playbook (SKILL.md) an agent follows step by step. This repo is the single canonical place where org-specific skills live — write one once, and Hermes, Cursor, Claude Code, Codex, GitHub Copilot and kagent can all run it.

The problem it solves

Before this repo, skills were scattered: .cursor/skills/ here, .claude/skills/ there, ~/.hermes/skills/ somewhere else — duplicated and drifting in every repo. The agentskills.io standard (Dec 2025) fixed the format: one folder layout, YAML frontmatter, progressive disclosure (discovery → activation → execution), and every major AI client knows how to load it. This repo is our implementation of that standard: consumers pull it in as a git submodule or a read-only mount and symlink the subfolders they need.

How each consumer loads it

ConsumerMechanism
Hermes (AI Agency)Read-only external skill mount in hermes-deployment/config/config.yaml — one of 7 mounted skill repos. Local agent-curated skills shadow registry skills on name collision.
Cursor / Claude Code / CodexSymlink the skills/ subfolders into each repo's client-specific skills dir (or reference directly — same folder format).
Admin portal copilotBuilt-in skills live in the portal's own catalog (see the Copilot Skills Guide); org skills from this registry are the model for cross-client playbooks.
Design rule: skills over integrations When the AI Agency needs a new capability, the order of preference is: (1) an existing MCP server + a skill that orchestrates it, (2) a new skill in this registry, (3) only as a last resort, runtime code in the fork. Skills are markdown — cheap to write, review and roll back.
